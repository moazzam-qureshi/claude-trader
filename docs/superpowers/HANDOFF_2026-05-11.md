```
   policy.yaml  (file seed + Tier 1 inviolable + Tier 2 fallback)
        │
        ▼
   settings.seed.bootstrap()  ──►  policy_settings table  (Tier 2 + Tier 3 rows)
                                          │
                                          ├──► settings.repo.get()        (read path)
                                          ├──► settings.repo.set_setting() (write path,
                                          │                                authority gate)
                                          ├──► settings.snapshot_policy()  (decision-time
                                          │                                snapshot)
                                          │
                              ┌───────────┼───────────┬──────────────┐
                              ▼           ▼           ▼              ▼
                         MCP tools   Discord       Discord       cli settings
                         (Claude)    /settings     /safety       (operator
                                     (Tier 3)      (Tier 2,      terminal)
                                                   operator-
                                                   only)
```

- Tier 1 (halal): file-only, NEVER mutable. Any path attempting
  mutation writes a rejected audit row with `rejection_reason='halal_inviolable'`.

  `operator_only_key`. Non-operator on `/safety` rejected with
  `not_operator` audit row.

  `/settings set` or MCP `set_setting`. Every mutation -> Discord
  notification.

preserving exact reproducibility without depending on `prompt_version
-> git checkout -> re-read policy.yaml`.
